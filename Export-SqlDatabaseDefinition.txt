
SQL DATABASE EXPORTER PROJECT PLAN
==================================

Project Goal
------------
Build an opinionated PowerShell tool:

    Export-SqlDatabaseDefinition

that:

- Exports a SQL Server database into a standardized folder structure
- Uses a YAML configuration file
- Produces Git-friendly output
- Exports security information
- Generates dependency documentation
- Creates a visual dependency map from sys.sql_expression_dependencies

This is NOT an SSMS Generate Scripts clone.
This is a Database-to-Folder Exporter with dependency documentation.

==================================================================

PHASE 2 - YAML CONFIGURATION
============================

[ ] Step 2 - Create export.yaml automatically

If export.yaml does not exist:

    Create default export.yaml
    Exit
    Tell user to review and rerun

This becomes the standard configuration used 99.9% of the time.

[ ] Step 3 - Add dependency settings to YAML

Settings should control:

- Dependency export enabled/disabled
- Include cross-database references
- Include cross-server references
- CSV output
- JSON output
- DOT output
- SVG output
- HTML report output

==================================================================

PHASE 3 - CONNECTION STRATEGY
=============================

[ ] Step 4 - Support SSMS launching

Preferred workflow:

    Open SSMS
    Connect to server
    Select database
    Run Exporter

Avoid requiring:

    -Server
    -Database

for normal use.

Support explicit parameters for automation later.

PHASE 8 - VISUAL DEPENDENCY OUTPUT
==================================

[ ] Step 19 - Generate Graphviz DOT file

        dependencies.dot

==================================================================

PHASE 9 - DEPENDENCY VISUALIZATION RULES
========================================

[ ] Step 22 - Keep Visualization Simple Initially

Version 1:

    Referencing Object --> Referenced Object

Example:

    dbo.vStudentSummary -> dbo.Student
    dbo.usp_LoadSecurity -> dbo.SecurityRole

[ ] Step 23 - Color-code later

Possible future enhancement:

- Tables = Blue
- Views = Green
- Procedures = Orange
- Functions = Purple
- External references = Red

[ ] Step 24 - Handle external references separately

Identify:

- Cross-database references
- Cross-server references

These are often migration risks.

[ ] Step 25 - Create dependency-warning report

    Dependencies\
        dependency-warnings.md

Include:

- Ambiguous references
- Caller-dependent references
- Cross-database references
- Cross-server references
- Unresolved objects

==================================================================

PHASE 10 - REFERENCE DATA
=========================

[ ] Step 26 - Export reference data

Controlled by YAML.

Use mainly for:

- StateCodes
- Campuses
- Security roles
- Other lookup tables

PHASE 11 - PROFILES
===================

[ ] Step 27 - Add named profiles

Examples:

- SchemaOnly

- DependencyMapOnly

PHASE 12 - GIT FRIENDLY OUTPUT
==============================

[ ] Step 28 - Make output deterministic

Same database should generate:

- Same file names
- Same ordering
- Same folder structure
- Repeatable output

Goal:

    git diff

shows real changes only.

[ ] Step 29 - Add git status summary

Future enhancement:

- Changed files
- New files
- Deleted files

No automatic commits initially.
